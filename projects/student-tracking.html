NorQuest Student Database — SQLite

Student & Grade Tracking | Academic Insights | SQL

Project File: View SQL Queries

Project Overview

This project presents a relational database system developed with SQLite to manage and analyze student information and academic performance within a hypothetical set of students in the NorQuest College machine learning program. The system supports operations such as updating records, retrieving filtered data, and performing performance-based aggregations.

The database facilitates administrative insights such as tracking Moodle access, analyzing performance trends, and filtering by enrollment dates.

Key Tables and Their Purpose

Table Description
Students Stores details about each student including their name, course enrollment, Moodle access status, date of registration, and notes.
Grades Contains grade information for various ML-related courses, linked to students using foreign keys.

Entity Relationship Design (ERD)

  • One-to-many relationship: A student can have multiple grades.
  • Grades include course name, score, and reference to student ID.
  • Grades.student_id is a foreign key referencing Students.student_id.

Data Initialization

  • 20 student records created
  • 30+ course grades logged
  • Included fields: moodle_access, course, score, registration_date, and notes
  • Constraints applied: NOT NULL, PRIMARY KEY, FOREIGN KEY

Key SQL Queries

  • Update Moodle access for repeat students
  • Filter students by course and enrollment date
  • Aggregate performance: average, max, and min scores by course
  • Count how many students passed (grades > 60%)

Business Insights

  • 35% of students scored over 80% in 3 or more courses
  • 25% scored below 60% in at least one course
  • 90% maintained Moodle access throughout the term

Recommendations

  • Initiate support programs for students below the pass threshold
  • Provide mentorship for repeaters and low-performers
  • Normalize course and grade data for future dashboarding
