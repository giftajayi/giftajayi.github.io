Conference Database Project

SQLite Relational Database | ERD | Attendee & Session Management

Project File: View SQL Queries & ERD

Project Overview

This project involves the creation of a relational database system using SQLite to manage a fictional conference. It includes defining the database schema, establishing relationships using foreign keys, inserting sample data, and executing SQL queries to derive insights. An Entity-Relationship Diagram (ERD) helps visualize the schema.

Key Tables and Their Purpose

  • Attendees: Stores information about conference participants including their name, email, and type (e.g., Student, Professional).
  • Sessions: Contains session details such as title, description, date, and time.
  • Speakers: Holds information about speakers and the sessions they are linked to.
  • Registrations: Links attendees to sessions, including payment status and timestamps.
  • Feedback: Stores post-session feedback and comments.

Entity Relationship Design (ERD)

  • One attendee can register for multiple sessions (via the Registrations table).
  • Each session can have multiple registrations and feedback entries.
  • Each speaker is assigned to a specific session.
  • Foreign key constraints ensure referential integrity across the database.

Sample Data Summary

  • 20 Attendees
  • 20 Sessions
  • 5 Speakers
  • 20 Registrations and Feedback records
  • Use of PRIMARY KEY, NOT NULL, and foreign key constraints

SQL Queries & Use Cases

  • List and filter attendees by type
  • Identify most attended sessions
  • Aggregate and analyze feedback per session
  • Generate personalized attendee participation reports

Insights & Business Metrics

  • Top Session: Session A with 6 attendees (30% of total)
  • Average Feedback: 1 entry per session
  • Demographics: Professionals make up 55% of attendees

Recommendations

  • Boost promotion for low-attendance sessions
  • Automate post-session feedback prompts
  • Use popular sessions as keynotes for future conferences
